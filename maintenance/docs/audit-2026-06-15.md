# Gearbox audit — 2026-06-15

Full-surface audit of the `gearbox` Claude Code plugin (fork `dividedby/gearbox`,
checked out at `~/repos/gearbox`, installed v0.1.7). Method: each phase delegated
to a tiered agent; findings synthesized here. Highest-stakes findings re-verified
directly against source before inclusion.

**Scope decisions (set with the maintainer):** documents only — no tracker
mutation (upstream is a quiet third-party repo we hold pull-only access to; the
fork has Issues disabled). Artifacts live here under `claude-config/gearbox/`. One
synthesis report (this file) + one roadmap (`roadmap.md`), not a per-domain doc
set — the repo is ~1,475 lines, mostly Markdown specs plus three Python scripts.

The unified, prioritized backlog is the table in [§7](#7-unified-backlog). The
forward plan keyed off it is `roadmap.md`.

## 0. Context & standing automation

- **What it is:** 5-agent tiered router (scout/grunt T0·haiku, builder T1·sonnet,
  architect T2·opus, verifier·haiku). A SessionStart hook injects the routing
  policy; a PostToolUse hook logs every `Task` delegation to
  `.claude/gearbox-log.jsonl`.
- **Real code:** `hooks/scripts/log-routing.py` (421), `bench/label.py` (267),
  `hooks/scripts/inject-routing.py` (45), and embedded bash in `commands/doctor.md`
  (243). Everything else is Markdown specs + JSON manifests.
- **Ground truth:** the maintenance `README.md` is current and internally
  consistent; synthesized from it rather than grilling.
- **Standing automation inventory:** **none.** No `.github/workflows`, no other CI.
  Every audit phase therefore ran in full — nothing is covered by an existing loop.

## 1. Staleness & hygiene

- **No CI.** Confirmed: no `.github/`, no other CI config.
- **Selfchecks present (good):** `log-routing.py --selfcheck` and
  `label.py --selfcheck` both pass (exit 0); cover the pure helpers including the
  falsy-coalescing and split-usage regressions. `inject-routing.py` has none (small
  enough to not need one).
- **Toolchain:** all three scripts are stdlib-only, Python 3.6+ (`#!/usr/bin/env
  python3`, type hints, f-strings; no walrus/`match`/`|`-unions). No
  `requirements`/`pyproject` — correct for a hook script set.
- **Versioning:** fork at `0.1.7`; **no GitHub releases on the fork** (upstream's
  latest *release* is v0.1.2, 2026-06-12). Fork ships via marketplace auto-update,
  not tags. A v1.0 bar wants real semver tags + a CHANGELOG (see roadmap).
- **No stale/dead files.** `bench/training-data.jsonl` is generated on first
  labeler run (absent now). `.claude/gearbox-log.jsonl` present and gitignored.

## 2. Architecture & code review

The design (named roles as a seam over model tiers; hooks for injection + logging)
is sound and appropriately small. Concrete code findings (verified at file:line):

- **`prompt_head` captures raw prompt text** (`log-routing.py:271`) and is copied
  onward by `label.py:66` into `bench/training-data.jsonl`. Security-relevant — see
  §4. **[G1]**
- **`_record_id` still collides for genuinely-parallel identical dispatches**
  (`label.py:21-37`): same agent + model + prompt within the same 1-second `ts`
  yields one id; the second is silently deduped at label time (`label.py:205-206`),
  undercounting the parallel-fan-out case the router is meant to learn from. **[G2]**
- **Log path from untrusted `event["cwd"]`** with no validation
  (`log-routing.py:289`); should prefer `CLAUDE_PROJECT_DIR` as `inject-routing.py`
  already does (`:19`). **[G3]**
- **Split-usage fallback requires both `input_tokens` and `output_tokens`**
  (`log-routing.py:124-131`) — a one-sided future payload silently yields
  `total_tokens=None`. **[G4]**
- **`bool` accepted as a token count** (`log-routing.py:116-119,148`):
  `totalTokens: True` → `1`. Latent data-quality trap. **[G5]**
- **String `<usage>` parse path reintroduces the falsy-`or` pattern**
  (`log-routing.py:163,176`) — a literal `0` in a usage string is skipped, unlike
  the (correct) dict path. **[G8]**

**Over-engineering (ponytail lens):**

- **Legacy key fan-out** across ~5 unobserved aliases per metric
  (`log-routing.py:116-146`) is speculative cross-version hedging; it is the bulk of
  the file's complexity. Trim to the empirically-confirmed keys + the documented
  split-usage fallback. **[G17]**
- **The string `tool_response` path** (`_tool_response_text` + `_parse_usage_string`,
  `log-routing.py:157-205`) handles a rendered-string response form never seen in
  15+ captured dispatches (always a dict). Dead weight — delete unless evidence
  surfaces. **[G18]**

## 3. Testing, quality & CI/CD

- Two runnable selfchecks exist and pass — a real strength for a repo this size.
- **Gap: no test that the *spec* is consistent with the *code*.** Nothing checks
  that `routing/routing.md`'s tier table, `_AGENT_ROUTING` in `log-routing.py:29`,
  and the `model:` frontmatter in each `agents/*.md` agree. This is exactly the
  drift that M3-class findings (§below) describe. **[G20]**
- **Gap: no CI.** Minimum bar: run both `--selfcheck`s + JSON/YAML lint of
  `hooks.json`/`plugin.json`/`marketplace.json` on push. **[G19]**
- `bench/tasks.md` is a manual benchmark *template*, not a runnable suite — the
  reward-signal pipeline has no way to measure whether a routing decision was
  *correct*, only that it happened. **[G25]**

## 4. Security & compliance

One material finding; the rest is solid.

- **[G1 · High] Secret leakage into a committable file.** `prompt_head` is the first
  200 chars of the raw delegation prompt (`log-routing.py:271`). It is gitignored in
  the log, but `label.py:66` copies it into `bench/training-data.jsonl`, which the
  4-pattern `.gitignore` (`__pycache__/`, `*.pyc`, `.claude/gearbox-log.jsonl`,
  `.DS_Store`) does **not** cover. A credential pasted into a delegation prompt can
  reach a committable artifact. *Fix:* gitignore `bench/training-data.jsonl`; reduce
  `prompt_head` to a hash/short slug or scrub secret patterns before write; stop
  propagating raw prompt text downstream.
- **Strengths (do not touch):** both hooks swallow all exceptions so a logger crash
  can never break the session (`log-routing.py:282-295`, `inject-routing.py:40-41`);
  JSONL line integrity is guaranteed (`json.dumps` escapes embedded newlines);
  `inject-routing.py` prefers `CLAUDE_PROJECT_DIR` with existence checks; doctor
  CHECK 8 fetches with a 5s timeout and SKIP-on-offline.
- **Minor hygiene:** `commands/doctor.md:156` `grep -l "@.claude/routing.md"` treats
  `.` as a regex wildcard — use `grep -F` and quote/`--`-guard file args. **[G6]**

## 5. Docs & reality reconciliation

Reference docs are accurate; the mismatches are spec-internal contradictions:

- **Verifier output format contradicts the reader.** `verifier.md:51` ("your reply
  must begin with the verdict line") vs `routing.md:62` ("scan … anywhere in it, not
  only line 1"). Tolerant reader vs strict writer — align them. **[G9]**
- **Verifier trigger wording drifts from the tier abstraction:** `verifier.md:3-4`
  says "after builder or architect" while `routing.md:48` says "T1/T2." **[G10]**
- **`_AGENT_ROUTING` tags verifier `T0`** (`log-routing.py:29`) but the tier table
  (`routing.md:10-16`) has no verifier tier — it's a meta-agent. Undocumented, will
  misattribute cost/tier in the learned-router data. **[G11]**
- **`plugin.json` author** still `Aditya Raj` (`:5-7`) after fork divergence — the
  fork-identity rules name only `repository`/`version` as expected divergences;
  decide author/contributors explicitly. **[G12]**
- **`init.md:29`** copies from `${CLAUDE_PLUGIN_ROOT}` with no unset-guard (doctor's
  CHECK 0 catches exactly this failure mode elsewhere). **[G7]**
- **Routing ambiguities:** "Max score 1-2 → T0" never says max-across-dimensions vs
  average (`routing.md:21-22`) **[G13]**; the T0→T2 "design problem" exception is
  undefined (`routing.md:27-28`) **[G14]**; the fallback rule names `Explore` as
  scout's built-in proxy (`routing.md:46`) — confirm it's a valid Claude Code
  subagent type or correct it **[G16]**; `ultrathink` is flagged experimental with
  no resolution path (`routing.md:72-74`) **[G15]**.
- **Architect has no Write/Edit** (`architect.md:4`); the spec says the parent routes
  execution to builder (`:14`), but the plan→execute handoff when architect is the
  circuit-breaker top tier (`routing.md:29-34`) isn't documented. **[G22]**

## 6. Prior art & competitive position

Crowded but winnable. Closest functional analog is
[`0xrdan/claude-router`](https://github.com/0xrdan/claude-router) (~41★, 39
releases) — rule + LLM-fallback classification across haiku/sonnet/opus, `/retry`
escalation; **no independent verifier, no reward-signal logging.** Larger repos
solve adjacent problems:
[`musistudio/claude-code-router`](https://github.com/musistudio/claude-code-router)
(~35k★) is an external proxy for non-Anthropic providers (manual, config-driven);
[`wshobson/agents`](https://github.com/wshobson/agents) (~37k★) is prescriptive tier
*documentation*, not automation. [RouteLLM](https://github.com/lm-sys/routellm) is
the learned-router reference but operates pre-call at the API layer.

- **Gearbox's genuine distinction:** named-role abstraction over tiers + independent
  verifier as a first-class post-edit step + JSONL telemetry explicitly framed as a
  learned-router reward signal. No single competitor combines these.
- **What competitors have that informs v1.0:** eval/benchmark harness (RouteLLM)
  **[G25]**; cost/escalation dashboard (`0xrdan`) **[G24]**; LLM-fallback
  classification **[G27]**; non-Anthropic escape hatch (`claude-code-router`)
  **[G29]**.
- **Existential risk:** Anthropic
  [issue #27665](https://github.com/anthropics/claude-code/issues/27665) consolidates
  30+ requests for native model routing. If shipped, it obsoletes the whole plugin
  tier — and gearbox's durable moat reduces to the verifier UX and the reward-signal
  data. The roadmap leans into those two. **[G31]**

## 9. Known unknowns

- **`Explore` proxy validity (G16)** — needs a live check that it's a real Claude
  Code subagent type before the fallback rule can be trusted.
- **`ultrathink` propagation (G15)** — unverified on the installed surface.

  dispatches; cost stays estimated, which caps the reward-signal quality (G26) and
  the learned router's ceiling (G27). Worth one more check on newer CC versions.
- **Native-routing timeline (#27665)** — the single biggest strategic uncertainty;
  governs how much to invest past v0.3.0.
