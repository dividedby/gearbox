# gearbox

## Language

- **Gearbox** — this plugin. Injects an automatic model-routing policy at SessionStart so the main session routes each piece of work to the cheapest tier that can do it well.
- **Orchestrator** — the main Claude session. Owns classification, routing, escalation, and verification; never the expensive default for trivial work.
- **Tier** — a cost/capability band a task is routed to. T0 (haiku), T1 (sonnet), T2 (opus). Routing is on the **max** of three dimensions, not the average.
- **Dimensions** — the three axes a task is scored 1–5 on: file scope, ambiguity, blast radius if wrong. The single highest score picks the tier.
- **scout** (T0) — read-only recon agent: search, read, summarize, "where is X / how does Y work".
- **grunt** (T0) — mechanical edits, 1–2 files, zero design decisions.
- **builder** (T1) — the workhorse: features, bug fixes, tests, refactors ≤5 files.
- **architect** (T2) — hard problems only: cross-cutting design, gnarly debugging, concurrency, migrations, security, performance. Read-only — returns a plan, does not edit or spawn subagents.
- **verifier** — read-only reviewer dispatched after any T1/T2 file-editing task; checks the diff against the task intent and returns `VERDICT: APPROVE` or `VERDICT: REJECT`.
- **Escalation ladder** — on a tier's failure or "needs escalation", move up exactly one tier with the full failure report; never retry a third time at the same tier; never skip T0→T2 without a design-level root cause.
- **Circuit breaker** — when the failing tier is already T2, stop and surface to the human rather than re-delegating blindly (one optional main-session attempt allowed).
- **Architect→builder handoff** — architect (read-only) returns a plan; the orchestrator dispatches a builder to execute it. Distinct from the circuit breaker.
- **Hard floors** — minimum tiers regardless of score: auth/payments/migrations/concurrency/secrets start at T1; production-breaking risk starts at T2.
- **Routing prior** — advisory table (`gearbox-recommendations.md`, regenerated by `/gearbox:recommend`) of historical tier/cost/approval rates. Tie-breaker only; never overrides hard floors or the circuit breaker.
- **Doctor** — `/gearbox:doctor`, the PASS/WARN/FAIL self-check of the installation. CHECK 8 reads `plugin.json` `repository` for freshness.
- **Routing log** — `~/.claude/gearbox-log.jsonl`, the per-project telemetry the plugin appends one line per dispatch; gitignored, not hand-maintained.
- **Baseline** — `.claude/gearbox-baseline.txt`, the pre-edit `git status --short` snapshot the `capture-baseline` hook writes before each T1/T2 dispatch for the verifier to diff against.
- **Fork identity** — this repo is a hard fork of `Adityaraj0421/gearbox` (abandoned). `plugin.json` `repository` points to `dividedby/gearbox` and `version` is the fork's own. See `maintenance/README.md`.
